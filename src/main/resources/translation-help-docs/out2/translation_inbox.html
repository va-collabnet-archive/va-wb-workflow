Translation inbox

The translation inbox displays the worklist items assigned and submitted for translation. It is divided in two parts:
  • The upper half of the left pane shows the Inbox containing the worklists assigned (with the numbers of worklist items in brackets). Also, there is another folder that displays the worklist items by status (delivered, to-do, rejected, etc.). In a third folder named “Custom”, tagged worklist items from different worklists and/or statuses are placed. Each user may create new tags according to his/her needs, priorities, etc.
  • Outbox shows in brackets the number of worklist items awaiting to be sent to the following step in the workflow:

  • The number of worklist items contained in the Inbox is displayed in brackets. This number will change as concepts are sent from the Inbox to the Outbox.
  • The number of worklist items in the Outbox will return to zero upon clicking Send button.

Pressing on 'Detach' opens an additional view of the worklist which also includes a filter option.

'Current item' displays the worklist item that is being displayed in the translation panel.

The lower part displays the worklist members in a table, according to the user’s specifications. Clicking on table headings will order worklist items alphabetically, in increasing or decreasing order.

Tagging worklist items

Worklist items can be tagged to indicate different user’s needs. Statuses will not change. Right clicking on a worklist item > Tag will display the available tags. Also a new tag can be created, specifying tag name, as well as it background color and text color

Tagged items will be displayed under Custom folder

Outbox (undo function)

All the concepts sent to the following step in the workflow are kept in a “transient outbox”. If any change is needed before submitting the concept to the next step, right clicking on the item will show an option to send it back to the inbox and place it in the original worklist. There, any changes required can be done. Worklist items cannot be changed directly in the Outbox. This is just an intermediate station from which items can be moved back into the inbox".

The Outbox will be automatically emptied when pressing “Send” button. At this point, translated items can no longer be recovered and they are moved to the next workflow step with status “Translated.”
